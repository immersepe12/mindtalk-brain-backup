# Proposal: Shift T13 Meta-Learner cadence to 8:30 PM IST Sunday (prevent Strategist write collision)

**Proposed:** 2026-06-28T20:00:00+05:30
**Source:** task13-meta-learner-2026-06-28
**Apply on:** 2026-07-05T20:00:00+05:30
**Status:** preview

## Issue detected

**Cron timing collision on Sundays:**

- T10 Strategist: `# Cadence: Every day at 8:00 PM IST`
- T13 Meta-Learner: `# Cadence: Sunday at 8:00 PM IST (after Learner at 6 PM)`

Both tasks run at 8:00 PM IST every Sunday. Strategist Step 10/11 applies Meta-Learner proposals (writes to `brain/proposed-changes/`, `brain/applied-changes/`, `brain/before-snapshots/`, `brain/BRAIN.md`, `brain/BACKLOG.md`). Meta-Learner writes NEW proposal files to `brain/proposed-changes/` and reads `brain/BRAIN.md`, `brain/BACKLOG.md`, `brain/WATCH.md` for its analysis.

If both run simultaneously:
- Both write to BRAIN.md / BACKLOG.md — one overwrite silently clobbers the other
- Strategist moves/renames proposal files while Meta-Learner is reading them → stale reads or file-not-found
- Meta-Learner reads BACKLOG.md in mid-write (half-written state) → inaccurate context

Evidence: BACKLOG.md 2026-06-27 Strategist stamp explicitly notes: *"3 Meta-Learner proposals eligible Sunday 06-28 at 20:04 IST — Sunday Strategist Step 10 will apply all 3."* This shows the Strategist is aware of the timing overlap and intends to handle it, but the overlap remains a structural race condition with no guard in either spec.

## Proposed change

### Part B — Cowork cron UI change (human-review-only, cannot be auto-applied)

Kushal must update the Cowork scheduled task `mindtalk-meta-learner` cron from `8:00 PM IST Sunday` to `8:30 PM IST Sunday` in the Cowork task scheduler UI. The task-spec comment change above (Part A) documents the intent; Part B makes it actually take effect.

**⚠ HUMAN-REVIEW-ONLY for Part B.** Strategist cannot auto-apply cron changes in the Cowork UI. Kushal applies Part B manually.

## Rationale

30 minutes is more than enough for Strategist Step 10/11 to: snapshot files, apply ≤3 proposals, rename them to `.applied.md`, move to `brain/applied-changes/`, update BRAIN.md/BACKLOG.md, and post Slack. Meta-Learner then starts with a clean state (all proposals from last week already applied, BRAIN.md reflecting today's Strategist updates). This eliminates the race condition and ensures Meta-Learner's analysis reads the most current state.

Without this fix, there's a recurring risk every Sunday that Meta-Learner reads a half-written BACKLOG.md or detects already-applied proposals as "still pending" (because Strategist is mid-rename).

## Risk assessment

Part A (doc change): near-zero. Cadence comment change only.
Part B (cron shift): low. 30-minute delay pushes Meta-Learner to 8:30 PM IST Sunday. No data dependency requires it to run at exactly 8 PM. Learner (6 PM) finishes well before either run.
